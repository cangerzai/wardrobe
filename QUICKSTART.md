# 快速开始指南

## 5分钟快速上手

### 第一步：打开项目
1. 打开微信开发者工具
2. 导入项目，选择本文件夹
3. 填写AppID（测试可以使用测试号）

### 第二步：添加图标（可选）
如果没有图标文件，TabBar会使用默认样式，不影响功能测试。

如需添加图标：
- 在 `images/` 目录下添加TabBar图标（参考 `images/README.md`）
- 或暂时注释掉 `app.json` 中的 `iconPath` 和 `selectedIconPath`

### 第三步：运行项目
1. 点击"编译"按钮
2. 在模拟器中查看效果

### 第四步：测试功能

#### 测试上传功能
1. 切换到"衣橱"页面
2. 点击"上传新服装"
3. 选择一张图片
4. 等待处理完成（约1.5秒模拟处理）

#### 测试换装功能
1. 上传至少2张图片
2. 切换到"首页"
3. 左右滑动切换服装
4. 观察卡通小人换装效果

## 项目结构说明

```
电子衣橱/
├── app.js              # 小程序入口，初始化云开发
├── app.json            # 全局配置，包含TabBar设置
├── app.wxss            # 全局样式
├── pages/              # 页面目录
│   ├── index/          # 首页：卡通小人+换装
│   ├── wardrobe/       # 衣橱：上传+列表
│   ├── discover/       # 发现：浏览分享
│   └── profile/        # 我的：个人信息
└── utils/              # 工具函数
    └── styleTransfer.js # 风格迁移服务
```

## 核心功能说明

### 1. 图片上传与处理
- 位置：`pages/wardrobe/wardrobe.js`
- 功能：选择图片 → 上传 → 风格迁移 → 保存
- 当前：使用模拟处理，需集成真实API

### 2. 卡通小人绘制
- 位置：`pages/index/index.js`
- 功能：Canvas绘制基础小人 + 服装叠加
- 交互：左右滑动切换服装

### 3. 数据存储
- 位置：使用 `wx.setStorageSync` / `wx.getStorageSync`
- 存储：本地存储，清除小程序会丢失数据
- 建议：生产环境使用云数据库

## 常见问题

**Q: 为什么看不到卡通小人？**
A: 确保已上传至少一张图片，且Canvas已正确初始化。

**Q: 图片处理很慢？**
A: 当前是模拟处理，实际API速度取决于服务提供商。

**Q: 数据会丢失吗？**
A: 当前使用本地存储，清除小程序数据会丢失。建议使用云数据库。

**Q: 如何部署上线？**
A: 1. 完善功能 2. 提交审核 3. 发布版本
