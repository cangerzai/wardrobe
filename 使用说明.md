# 使用说明（上传发布前）

本文档面向当前项目结构，按“本地开发 -> 云函数部署 -> 小程序上传”流程说明。

## 一、微信开发者工具配置

1. 打开项目目录：`电子衣橱`
2. 确认 `project.config.json` 的 `appid` 正确
3. 确认已开通云开发，且云环境与代码配置一致

## 二、Python 服务配置（卡通化与姿态检测）

项目后端在 `main.py`，提供：
- `POST /cartoonize`
- `POST /pose`
- `GET /health`

### 1) 安装依赖

```bash
pip install -r requirements.txt
```

### 2) 启动服务

```bash
uvicorn main:app --host 0.0.0.0 --port 8000
```

### 3) 模型文件检查

确保路径存在：`models/animeganv3.onnx`

## 三、云函数部署

目录：
- `cloudfunctions/getOpenid`
- `cloudfunctions/styleTransfer`
- `cloudfunctions/toggleLike`

### 推荐方式（避免云端安装依赖报错）

在每个云函数目录执行 `npm install`，然后在开发者工具中上传“所有文件”。

> 如果你的机器提示 `npm 不是内部命令`，先安装 Node.js LTS 并重开终端。

## 四、关键联调项

### 1) styleTransfer 云函数地址

文件：`cloudfunctions/styleTransfer/index.js`

确认：
- `CARTOON_API_BASE_URL` 指向可访问的 Python 服务

### 2) 前端服务地址

文件：`pages/index/index.js`

确认：
- `CARTOON_API_BASE` 与后端实际地址一致（用于 `/pose`）

## 五、上线前功能回归

请至少验证以下流程：

1. 衣橱上传一件上衣、一件裤子
2. 首页可切换并叠加展示
3. 长按修改分类后，切页返回仍保持
4. 备注修改后，切页返回仍保持
5. 删除服装后，立即从衣橱消失且不会“回弹”
6. 发现页、我的页能正常打开

## 六、常见问题

### Q1：云函数安装依赖报 `ret:-3 system error`
优先改为本地 `npm install` 后上传“所有文件”。

### Q2：上传后首页没显示新衣服
检查当前筛选分类（上衣/裤子）是否一致。

### Q3：服装位置与新模特不贴合
在 `pages/index/index.js` 调整：
- 上衣：`topY/topBottom/topW`
- 裤子：`basePantsY/pantsW` 与 `waistOffset` 补偿系数
